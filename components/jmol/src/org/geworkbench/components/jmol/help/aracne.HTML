Aracne

This help section describes using geWorkbench to run ARACNE (Algorithm for the Reconstruction of Accurate Cellular Networks). ARACNE is an information-theoretic algorithm used to identify transcriptional interactions between gene products using microarray expression profile data. The details of the algorithm are described in Margolin et al., Nature Protocols (2006), Vol 1 (2) p663-672 for a full description of this code and interface (http://www.nature.com/nprot/journal/v1/n2/abs/nprot.2006.106.html).

Run Arcane Analysis

1. In the Project Panel, load a microarray dataset or adjacency matrix.

2. Populate the Aracne Parameters which includes:

。 Hub Marker (disabled for adjacency matrix) by (1) explicitly typing the marker name or (2) clicking Load to load a .txt file listing the Hub Markers from an external source. This text file should contain one probe identifier in each line which corresponds to the name of a probe in the input data file.

。 Update the value of the Threshold Marker or accept the default values.

。 Kernel Width: (disabled for adjacency matrix) The kernel width used for the computation of the MI values.

。 DPI Tolerance: If the Do Not Apply option is specified, no DPI is applied. If Apply is selected then the DPI is applied and the value (between 0 and 1) in the associated text box is used to determine the stringency level of the DPI application.

。 DPI Target List: If the DPI Target List checkbox is selected, markers can be specified by (1) explicitly typing the marker name or (2) clicking Load to load a .txt file listing the Markers from an external source which will be used when the DPI is applied. Specifically, the DPI will be applied to an edge (A, B) of the adjacency matrix only if at least on of the markers A, B belong to the DPI target list.

。 All Markers: (disabled for adjacency matrix) Specifically, if the “All Markers” checkbox is not selected then activated markers are eligible to participate in MI computations; otherwise, all markers in the microarray set are eligible.

。 All Arrays: (disabled for adjacency matrix) If the “All Arrays” is not selected then only values from activated microarrays participate in the computation of the MI values. Otherwise values from all microarrays are used.

3. Select where to run the analysis in the Services subtab. Local runs an instance of the ARACNE server running on the user’s machine and Grid allows the user to pick up one among the available caGrid-enabled versions of ARACNE.

4. Click Analyze. A progress indicator is displayed to reflect the analysis status. If successfully, the progress window will close and an adjacency matrix result child node is included in the project panel. This result node is displayed in the Cytoscape visualization. Refer to Cytoscape help for a full description of Cytoscape.

Note: The adjacency matrix (result node) Dataset History will reflect the parameters used to the analysis.
