Pattern Discovery

The Pattern Discovery is located in the Selection Area in the lower left of the application.  This component runs SPLASH,  an efficient application that identifies sparse amino acid or nucleic acid patterns (called "motifs") that match identically or similarly in a set of protein or DNA sequences. For a full description of SPLASH, go to http://www.research.ibm.com/splash/index.htm.

SPLASH is hosted on a remote server. The caWorkbench component communicates with the server via a SOAP interface. Search parameters and protein/DNA sequences in FASTA file format are uploaded to the server via SOAP. SPLASH is run and the matching sequences are retrieved and displayed in the selection area.   This section describes the basic usage of the pattern discovery panel, modes of pattern discovery and Pattern Discovery Results.

Basic Usage

The steps of basic usage are indicated in the screenshot above.

A.      The modes of Pattern Discovery are: Normal Discovery, Hierarchical Discovery and Exhaustive Discovery. Across the top of the Pattern Discovery panel are three radio buttons (Norm., Hierarch., and Exhaust.) that are used to select the discovery mode.

B.      The default "Basic" parameters are shown. The Parameters help section includes a full description of the Pattern Discovery parameters.

C.      Specifies use of a Globus (http://www.globus.org/toolkit/ ) instance of SPLASH for Pattern discovery. This allows for discovery of SPLASH instances running on the internet in accordance with the Globus Alliance’s requirements (server discovery not fully implemented). If unselected, a GSOAP (http://gsoap2.sourceforge.net/ ) implementation of a SPLASH web service hosted at Columbia University is used.

D.      Clicking on the clockwise blue arrow will Start Pattern Discovery. This brings up a New Session dialog box. The first time Pattern Discovery is run, the dialog box must be filled out. Use the drop down menu to select the Server Type and Server that hosts SPLASH. Then type-in the port and User Name. The image below is a connection to a server running on the local host at port 80.

      Available SOAP SPLASH server(s) are at splash.cu-genome.org at port 80. Please select ‘splash’ as the server type.

E.       Clicking on the stop sign icon will Stop Pattern Discovery.

F.       Clicking on the folder icon will Load a Pattern File motif files (.pat).

G.      The progress bar used to provide a visual update of the percentage completion.

Pattern Discovery Results

Shown below are the results of a discovery using default parameters.  The Discovery Table is filled with information on the discovered motifs. The steps of basic usage are indicated with letter in the below screenshot.

A.      The number of patterns found is displayed below the discovery table.

B.      Right clicking on a pattern displays the option to:

➺       Mask pattern- suppresses the selected pattern from the set of patterns discovered. So the sequence panel for instance will not show a masked pattern match in one of its modes of operation and will exist in any ancillary dataset created (see below).

➺       Unmask all patterns- causes all patterns discovered to be “active” and these will be shown in any ancillary dataset created.

➺       Save patterns- caves the regular expression corresponding to the selected pattern to file.

➺       Save selected patterns- save the selected patterns to file along with match coordinates on sequences on which the pattern discovery was performed.

➺       Save All patterns- save all patterns to file along with match coordinates.

➺       Add pattern to project- creates an ancillary dataset in the Project Panel under the sequence dataset used for pattern discovery.

C.     To display selected motifs and their frequencies in the dataset, left click on a pattern(s).  The selected pattern(s) are then shaded gray in the discovery table and are displayed graphically in the Sequence Panel, Promoter Panel and Patterns.

Modes of Pattern Discovery

Normal Discovery

Normal discovery is based on the algorithm SPLASH (Califano, A., 2000); it generates a list of all regular expression patterns (motifs) that satisfy a user-defined minimum support and a minimum density criteria.

Hierarchical Discovery

Hierarchical discovery is used to analyze a sequence database and produce a sequence of clusters based on discovered patterns. For more information, visit http://www.research.ibm.com/splash/Hyerarchical/HierarchicalDiscovery.htm.

Exhaustive Discovery

Exhaustive discovery can be used to exhaustively analyze a sequence database for all non-overlapping motifs that are statistically significant. This is useful for discovering in order of relative sequence support, all regions of a protein family that have been preserved by evolution and may therefore play a functional or structural role. For more information, visit http://www.research.ibm.com/splash/Exhaustive/ExhaustiveDiscovery.htm.
