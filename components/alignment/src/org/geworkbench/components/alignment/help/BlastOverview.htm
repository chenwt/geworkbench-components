Blast and Other algorithms

The application offers some popular sequence alignment algorithms and a simple Blast Parser.

The Paracel Blast Machine is used as default backend blast Machine, GeneMatcher2 are used for other algorithms. Both Machines are hosted at Columbia Genome Center.

To check the status of Blast Machine, use the "Blast_Server_Info" panel under the Blast_Parameters module, click Connection button to check the status:

To run your job: First load a FASTA file into project, then you can view the Basic setup, choose which database and blast program to use. Then click "BLAST" Button.

It will automaticly check Blast Machine status first, then if the sever is listening, the job will be submitted to the server.

The programs include in blast are:

  • blastp: compares an amino acid query sequence against a protein sequence database
  • blastn: compares a nucleotide query sequence against a nucleotide sequence database
  • blastx: compares a nucleotide query sequence translated in all reading frames against a protein sequence database
  • tblastn: compares a protein query sequence against a nucleotide sequence database dynamically translated in all reading frames
  • tblastx: compares the six-frame translations of a nucleotide query sequence against the six-frame translations of a nucleotide sequence database.

 

If you want to use other algorithms, you can select from "Other Algorithms" panel:

  • smith waterman Protein : compares an amino acid query sequence against a protein sequence database
  • smith waterman DNA : compares a nucleotide query sequence against a nucleotide sequence database
  • frame(from DNA sequence to protein DB): compares a nucleotide query sequence against a protein sequence database
  • frame(from Protein sequence to DNA DB): compares an amino acid query sequence against against a nucleotide sequence database

 

 

All other algorithms requests will be processed on GeneMatcher2 machine.

 

You can also use "advanced" panel l to set up more alignment options. In default, Your result will be showed both on an new web browser window and the "Blast Parser" Module. You can diselect this option.

 

 

Frame Shift Penalty and Query generic code are optional selection used for blastx.

After the job is submitted, you can see the moving of the progress bar. If you want to stop the running job, click the "stop" button.

 

 

The result will be showed on the BlastParser module and based on your choice, it may show on a new web browser too.

 

You can pick the hits you are interested, and collect the selected sequences of those hits into project. The program will generate a temparary FASTA file for those sequences, a new file node will be added under currrent project. You can save them later.
