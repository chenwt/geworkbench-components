Blast and Other Algorithms

The application offers some popular sequence alignment algorithms and a simple Blast Parser. The Paracel Blast Machine is used as default backend blast Machine, GeneMatcher2 are used for other algorithms which are both hosted at Columbia Genome Center. The Alignment Results help section describes the Blast results.

This help section covers: checking blast server status, running a Blast job, Advanced Options, and using other algorithms.

Checking the status of Blast Machine

1 Select the Sequence Alignment tab in the Analysis Area in the lower left of the application.

2 Click on the Service tab.

。 Default Columbia : By default, jobs will be submitted to the BLAST server at Columbia. This server runs Paracel BLAST and is optimized for large searches.

。 NCBI: This has the advantage of providing access to the latest version of the NCBI BLAST program.

Note: The text field at the bottom shows that one sequence has been selected. If you have a Fasta file that has multiple sequences, you can select the ones you want in the Markers component and activate this selection, letting you search on a subset.

You may search on all the sequences in a file by clicking the All Markers checkbox.

3 Click on Refresh to check the status of the service.

Running a Blast

1 Load a FASTA file into a project.

2 In the Analysis Area, select the Sequence Alignment tab.

3 Select the Blast tab.

4 Select program from the dropdown list. This includes:

。 blastp: Compares an amino acid query sequence against a protein sequence database.

。 blastn: Compares a nucleotide query sequence against a nucleotide sequence database.

。 blastx: Compares a nucleotide query sequence translated in all reading frames against a protein sequence database.

。 tblastn: Compares a protein query sequence against a nucleotide sequence database dynamically translated in all reading frames.

。 tblastx: Compares the six-frame translations of a nucleotide query sequence against the six-frame translations of a nucleotide sequence database.

5 Modify the values of subsequence if needed.

6 Click on the database name to select the Database.

7 Click on the curved arrow submit button. It will automatically check Blast Machine status first, then if the sever is listening, the job will be submitted to the server.

8 After the job is submitted, the progress bar will indicate that BLAST is now running. If you want to stop the job, click Stop (red hexagon).

9 The result will be showed on the BlastParser module and based on your choice; it may show on a new web browser. You can unselect “Display your result in your browser”, which is located under Advanced Options. The Alignment Results help section describes the Blast results.

    Other Algorithms

    To use other algorithms, select from "Other Algorithms" panel. All other algorithms requests will be processed on GeneMatcher2 machine.

    1 Load a FASTA file into a project

    2 In the Analysis Area, select the Sequence Alignment tab

    3 Select the Other Algorithms tab

    4 Select a Database from the dropdown list.

    5 Modify the values of subsequence if needed

    6 Select an Algorithm from the dropdown list. The includes:

    。 Smith-Waterman Protein: compares an amino acid query sequence against a protein sequence database

    。 Smith-Waterman DNA: compares a nucleotide query sequence against a nucleotide sequence database

    。 Frame(from DNA sequence to protein DB): compares a nucleotide query sequence against a protein sequence database

    。 Frame(from Protein sequence to DNA DB): compares an amino acid query sequence against a nucleotide sequence database

    7 Click on Search.

    。 The results are displayed in the Alignment Results visualization component.
