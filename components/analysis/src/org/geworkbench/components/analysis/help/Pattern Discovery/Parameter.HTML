Setting Parameters

Pattern Discovery results can be tweaked by setting parameters used to conduct the search on the SPLASH server. The parameters are set via SOAP on the server instance of SPLASH before the actual discovery. Some parameters depend on the kind of search conducted (basic, exhaustive or hierarchical). Parameters are set in the pane at the bottom of the pattern discovery module. A full description of the capabilities and functionality of SPLASH can be found at http://www.research.ibm.com/splash/index.htm . The following parameters can be applied to discovery modes: Limits pane, Grouping or Advanced.

Basic Pane

Although the default pane is labeled basic, its values are used in all three discovery modes. The meaning of these values is as follows:

。 Support: restricts pattern discovery to patterns that occur in at least the set percentage or decimal value of the total number of sequences in the input file. Note that this can be expressed as a percentage value (default 70%) or decimal value between 0 and 1.

。 Min Tokens: reported values must have at least the number of tokens set by this value.

。 Density Window: the maximum window length for each substring that starts with a token.

。 Density Tokens: the min number of matching tokens within a window.

Hierarchical Pane

The Hierarchical pane is used to set parameters during a hierarchical discovery. These parameters are:

。 Min. Cluster Size: largest cluster size for which no further analysis is performed.

。 Min. Pattern Number: support and density are decreased until at least min patterns are found that satisfy all other constraints.

Below is the hierarchical pane with default parameters set.

Exhaustive pane

The Exhaustive pane is used to set parameters during an exhaustive discovery. These parameters are:

。 Min. Support: lowest motif support as a percentage of the total number of sequences in the input file.

。 Min. Pattern Number: Support and density are decreased until at least min patterns are found that satisfy all other constraints.

。 Dec. Support: Percentage by which to decrease min support on the next iteration.

。 The disabled portions are not implemented yet.

Here’s exhaustive pane with the defaults set.

Limits pane

The Limits pane is used to define boundaries on the pattern discovery. The parameters are:

。 Max. Pattern Number: The maximum number of patterns that can be found before the discovery is terminated.

。 Max. Run Time (sec): The maximum amount of time in seconds that the discovery can be run before it is terminated.

Grouping

The Grouping pane denotes the grouping type for the discovery.

Advanced

The Advanced pane rounds up the last set of parameters. It defines the following set of options:

。 The drop down box sets the name of the similarity matrix. The similarity matrix is used to define similarity classes.

。 Similarity Threshold: This option sets the threshold used to define similarity classes.

。 ZScore: reported patterns must have a zscore at least equal to this value.
