Normalizers

The application offers a selection of pluggable normalizers to assist in the manipulation of the data.

Normalizers are used to perform some needed data transformation, usually in preparation of an analysis step to follow.

Normalizing a dataset A will give rise to a new dataset B. The dataset B will appear as a child of A in the project tree window. Available normalizers are displayed in the Normalization Panel (figure below). Experienced users can add or remove normalizers by manipulating the relative entries in the application configuration file (see the design documentation). Most normalizers have associated parameters that control their operation. Values for these must be provided in the appropriate areas within the Normalization Panel.

The application comes configures with the following normalizers:

Normalizers

  • Missing value computation: Replaces every "missing" value with either the mean value of its marker across all microarrays, or with the mean measurement of all markers in the microarray where the missing value belongs.

  • Log transformation: The log2 transformation is applied to all measurements in a microarray.

  • Threshold value: All data points whose value is less (or more) than a user-specified minimum (maximum) value are raised (reduced) to that minimum (maximum) value.

  • Marker-based centering: Subtracts the mean (median) measurement of a marker profile from every measurement in the profile.

  • Microarray-based centering: Subtracts the mean (median) measurement of a microarray from every measurement in that microarray.

  • Mean-variance normalizer: For every marker profile, the mean measurement of the entire profile is subtracted from each measurement in the profile and the resulting value is divided by the profile’s deviation.
